# 人才管理系统 · 腾讯云 CloudBase 部署指南（傻瓜版）

> 目标：把系统跑在腾讯云上，拿到一个公网网址，同事点开就能用。
> 全程在网页上点，不用装软件、不用碰代码、不用碰 GitHub（已为您绕开）。

---

## 第 0 步：准备（一次性）
1. 打开 👉 https://cloud.tencent.com 用**微信扫码**注册腾讯云。
2. 按提示做**实名认证**（手机号 + 偶尔刷脸）——国内平台都这个流程，几分钟。
3. 进入「云开发 CloudBase」控制台 👉 https://console.cloud.tencent.com/tcb
4. 点「新建环境」，地域选**上海**（或离您近的），环境名随意（如 `hr-env`），按提示开通。
   - 云开发有免费额度，试跑基本不花钱。

---

## 第 1 步：新建云托管服务
1. 在左侧菜单点 **「云托管」** → **「新建服务」**。
2. 服务名称随意，如 `hr-talent`。
3. **部署方式**选下面任意一种（推荐方式甲，最省事）：

### 方式甲：通过「公开 Git 仓库」部署（推荐，不用下载任何东西）
- 仓库地址填：`https://github.com/Tongcan-Xu/hr-talent-system`
- 分支：`main`
- 端口：`3000`
- 目标目录：留空（默认根目录）
- 点「创建并部署」

### 方式乙：通过「本地代码」上传（备选，如果上面那个选项找不到）
- 用我给您打包好的 `hr-talent-deploy.zip`，在「上传代码包」处选它上传即可。
- （这个 zip 我已经帮您打好了，里面是完整的系统代码 + Dockerfile）

## 几个提醒（重要）
1. **改默认密码/密钥**：上线后请第一时间把管理员密码和公司密钥改成内部专用（避免外人用默认值注册进来）。要改告诉我，我帮您设。
2. **数据库在海外**：当前数据库用的是免费的 Neon 云库（在美国），国内访问可能略慢（几百毫秒），试跑完全够用。等效果满意、要正式用时，可以换成腾讯云自家的 PostgreSQL（国内、快），**只改一个 `DATABASE_URL` 就行，数据一条不丢**。
3. **休眠**：云托管默认"低成本模式"会空闲缩容，下次打开可能慢几秒（冷启动）。试跑无所谓；要随时秒开可在服务设置里改"高可用模式"（可能计费）。
4. **迁移到公司内网**：将来要搬回公司服务器，代码和数据库都不用改——把同一个程序指到同一个 `DATABASE_URL` 即可（详见之前说明）。
