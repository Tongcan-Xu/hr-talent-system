### 路线 B（全部自己点 · 不依赖我）：网页 Dashboard 一步步来

**第 1 步：拿到数据库地址（Neon）**
1. 打开 https://neon.tech 注册（免费、不要信用卡，可用 GitHub 登录）
2. 点 **Create Project**（新建项目），区域选 **AWS US East (Ohio)** 或离您近的
3. 创建后会看到一串数据库连接串（Pooled connection string），**复制保存**——形如：
   `postgresql://neondb_owner:xxxx@ep-xxxx-pooler.xxxx.aws.neon.tech/neondb?sslmode=require`
   （这就是要填给系统的 `DATABASE_URL`）

**第 2 步：把系统跑成公网网址（Render）**
1. 打开 https://render.com 注册（免费、不要信用卡，可用 GitHub 登录）
2. 登录后，点页面**右上角「+ New」** → 选 **Web Service**
   （注意：不是 "Create App"，是 **+ New → Web Service**）
3. 连接 GitHub：首次会跳授权，点 **Authorize Render**，允许它访问您的仓库
4. 仓库列表里选 **Tongcan-Xu / hr-talent-system**，分支 **main**
   （页面若提示检测到 `render.yaml` 自动配置，直接采用即可）
5. 实例类型选 **Free**（免费）
6. 展开 **Environment Variables（环境变量）**，添加 3 项：
   | 名称 | 值 | 说明 |
   |------|-----|------|
   | `DATABASE_URL` | 第1步复制的那串 Neon 连接串 | 数据库地址 |
   | `COMPANY_KEY` | 自定义，如 `hualian2026` | 同事注册时要用 |
   | `ADMIN_PASSWORD` | 自定义管理员密码 | 替换默认的 admin123 |
7. 点 **Deploy**，等 1~3 分钟，状态变 **Live / Healthy** 即可
8. 在服务详情页复制 **URL**（形如 `https://hr-talent-system-xxxx.onrender.com`）
   —— 这就是给同事的公网链接

1. 🔐 **上线后立刻改密钥/密码**：公司密钥 `COMPANY_KEY` 和 管理员密码 `ADMIN_PASSWORD` 默认是示例值，外人可能猜到。请在 Render 环境变量里改成只有内部知道的。

## 五、常见问题

- **打不开 / 一直转圈？** 多半是休眠中，等 1 分钟再刷；若长期不行，检查 Render 里 `DATABASE_URL` 是否填对、`render.yaml` 是否自动识别。
- **数据没了？** Neon 免费库不会丢数据；若看到空库，是换了数据库地址（DATABASE_URL 填错或换了库）。
- **想换平台？** 代码已同时支持 SQLite / Postgres，换任何支持 Node 的平台都能跑。
